# Final Source of Truth for Slides and Report

This file summarizes the project as implemented in the current GitHub repo and the latest rerun. 

The final slide figures are the `_split.png` figures generated by `07_figures_split.py`.

## Current Repo Sources

Primary files:

- `01_collect.py`: collects Polymarket Gamma market metadata and CLOB price history.
- `02_clean.py`: applies the cleaning filters.
- `03_snapshots.py`: extracts fixed-horizon snapshots.
- `04_label.py`: labels usable markets by genre with a 500-per-genre cap.
- `05_analyze.py`: computes calibration tables, Brier decomposition, and regression checks on the labeled data.
- `07_figures_split.py`: creates the final split figures used in the deck, including the crypto split and final split-figure preprocessing.
- `figures/figure1_reliability_split.png`: final calibration/reliability diagram.
- `figures/figure2_brier_comparison_split.png`: final mean Brier score chart.
- `figures/figure3_brier_decomposition_split.png`: final Brier decomposition chart.
- `figures/figure4_accuracy_over_time_split.png`: final accuracy-over-time chart.


## Core Research Questions

Use these three research questions:

1. Are Polymarket probabilities calibrated?
2. Does forecast performance vary by genre?
3. Does performance change as markets approach resolution?

## Pipeline

The implemented pipeline is:

`Collect -> Clean -> Label -> Snapshot -> Score -> Visualize`

Details:

- `01_collect.py` collects market metadata from Polymarket's Gamma API and price history from the CLOB API.
- `02_clean.py` removes markets that are not clean binary forecasting targets.
- `04_label.py` labels markets by genre, using category tags first and keyword matching as fallback.
- `03_snapshots.py` extracts probabilities at fixed horizons before resolution: `1h`, `12h`, and `1d`.
- `05_analyze.py` computes calibration, Brier score decomposition, and logistic regressions.
- `07_figures_split.py` creates the current split figures used for final slides.
